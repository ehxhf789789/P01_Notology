## Notology란?

Notology는 구조화된 노트 템플릿, 위키링크 연결, 강력한 검색 엔진을 하나의 네이티브 앱에 결합한 **데스크톱 지식 관리 앱**입니다.

**핵심 원칙:**

- **플레인 마크다운 파일** &mdash; 독점 형식 없음, 종속성 없음
- **휴대 가능한 보관함** &mdash; 보관함은 그냥 폴더입니다. USB, NAS, 클라우드 동기화 폴더에 넣고 어디서든 사용하세요
- **타입이 있는 노트** &mdash; 12가지 템플릿 (회의, 논문, 연락처 등)으로 빈 페이지가 아닌 구조화된 노트 작성
- **네이티브 성능** &mdash; Rust 백엔드 (Tauri v2) + React 프론트엔드. Electron 오버헤드 없음
- **오프라인 우선** &mdash; 계정 없음, 구독 없음, 인터넷 불필요

## 누구를 위한 앱인가요?

Notology는 지식 집약적인 모든 워크플로우에 적응합니다. 실제 활용 사례를 소개합니다:

### 작가 & 스토리텔러

<!-- 스크린샷: "Characters/", "Locations/", "Plot Arcs/" 폴더가 있는 폴더 트리, -->
<!-- CONTACT 템플릿을 캐릭터 시트로 활용 (이름, 특성, 관계 프론트매터), -->
<!-- 그래프 뷰에서 캐릭터-장소 간 위키링크 연결 표시. -->
<!-- ![작가 활용 사례](docs/images/usecase-writer.png) -->

소설의 세계관을 직관적으로 관리하세요. **CONTACT 템플릿**을 캐릭터 시트로 활용하여 성격, 배경, 관계를 구조화된 필드로 정리할 수 있습니다. **폴더 노트**로 장소, 플롯 아크, 타임라인을 구성하세요. **그래프 뷰**는 캐릭터, 사건, 배경 사이의 숨겨진 연결을 드러내어 스토리 세계의 전체 조감도를 제공합니다.

> *예시 보관함 구조:*
> ```
> 나의 소설/
> ├── 등장인물/
> │   ├── 등장인물.md              (폴더 노트: 캐릭터 인덱스)
> │   ├── CONTACT_아리아.md        (캐릭터 시트)
> │   └── CONTACT_마르쿠스.md
> ├── 장소/
> │   ├── 장소.md
> │   └── NOTE_오래된 도서관.md
> ├── 플롯 아크/
> │   ├── NOTE_1막 - 부름.md
> │   └── NOTE_2막 - 여정.md
> └── 자료 조사/
>     ├── LIT_중세 건축.md
>     └── DATA_역사 연표.md
> ```

개인 연구 라이브러리를 구축하세요. **PAPER 템플릿**으로 읽은 논문마다 DOI, 저자, 초록을 체계적으로 기록합니다. `[[위키링크]]`로 논문 간 아이디어의 계보를 추적하세요. **호버 윈도우**로 PDF를 읽으면서 동시에 노트를 작성할 수 있습니다. **THEORY 템플릿**으로 가설을 탐구하고, **DATA 템플릿**으로 데이터셋과 방법론을 문서화하세요.

- **문헌 리뷰**: LIT 템플릿으로 출처, 핵심 논점, 비판적 노트를 기록
- **교차 참조**: 논문, 이론, 데이터 간 위키링크가 상호 연결된 지식 베이스를 형성
- **즉각적인 검색**: Tantivy 전문 검색으로 수천 개의 노트에서 논문, 인용문, 개념을 밀리초 내에 검색

일상 업무와 개인 지식 관리를 효율화하세요. **MTG 템플릿**이 참석자, 안건, 액션 아이템이 포함된 회의록을 자동 생성합니다. **캘린더 뷰**로 마감일을 추적하세요. 컨테이너로 프로젝트를 정리하고 **5가지 검색 모드**로 무엇이든 즉시 찾을 수 있습니다. **메모 시스템**으로 원본을 수정하지 않고 문서에 주석을 달 수 있습니다.

- **회의록 작성**: MTG 템플릿의 날짜 접두사 파일명으로 시간순 정리
- **프로젝트 정리**: 프로젝트별 폴더, 폴더 노트를 프로젝트 대시보드로 활용
- **휴대 가능한 업무 환경**: NAS에 보관함을 두고 사무실, 집, 이동 중 어디서든 접근

<!-- (정보, 팁, 경고 타입)으로 된 학습 노트, 과목 간 개념을 연결하는 위키링크, -->
<!-- 도메인 태그별로 색칠된 지식 네트워크를 보여주는 그래프 뷰. -->
<!-- ![학생 활용 사례](docs/images/usecase-student.png) -->

모든 과목의 학습을 체계화하세요. **SEM 템플릿**으로 발표자와 주제 필드가 포함된 강의 노트를 작성합니다. **위키링크**로 과목을 넘나드는 개념 맵을 만드세요 &mdash; 물리 개념을 수학 유도과정에, 수학을 실험 보고서에 연결합니다. **그래프 뷰**는 지식이 어떻게 연결되는지 보여주어, 시각적 연상을 통해 기억을 강화합니다.

- **학습 노트**: 콜아웃 블록, 코드 블록, 수식 표기가 가능한 마크다운 서식
- **시험 준비**: 전체 노트 대상 전문 검색 &mdash; 특정 개념이 언급된 모든 곳을 즉시 검색
- **지식 매핑**: 그래프 뷰로 어떤 주제가 잘 연결되어 있고 어떤 주제를 더 공부해야 하는지 파악
- **휴대성**: USB 드라이브 하나에 전체 학습 보관함을 넣어 집, 학교, 도서관을 오가며 사용

## Notology vs Obsidian

Notology는 Obsidian의 보관함 기반 접근과 위키링크 철학에서 영감을 받았습니다. 차이점은 다음과 같습니다:

| | **Notology** | **Obsidian** |
|--|-------------|-------------|
| **엔진** | Tauri v2 (Rust + WebView) | Electron (Chromium) |
| **검색** | Tantivy 전문 검색 엔진 (Rust) | 내장 파일 검색 |
| **노트 타입** | 12개 구조화된 템플릿 + 자동 프론트매터 | 빈 마크다운 + 커뮤니티 템플릿 |
| **멀티 윈도우** | 내장 호버 윈도우 (드래그, 스냅, 리사이즈) | 팝아웃 윈도우 (별도 OS 창) |
| **캔버스** | 플로우차트 도형 (다이아몬드, 평행사변형, 원) + 화살표 | 카드와 연결선 |
| **코멘트** | 내장 노트별 코멘트/메모 시스템 | 플러그인 필요 |
| **그래프** | 물리 제어 + 타입별 컬러링 | 내장 그래프 |
| **휴대성** | 보관함 잠금 + 충돌 감지 (공유 드라이브 지원) | 유료 Sync 서비스 |
| **가격** | 무료 오픈소스 | 코어 무료, Sync/Publish 유료 |
| **코드** | 오픈소스 (MIT) | 비공개 |

- **커스텀 키보드 단축키** &mdash; 모든 동작 재매핑 가능

빈 페이지 노트 앱과 달리, Notology는 구조화된 프론트매터가 포함된 **타입이 있는 노트**를 제공합니다.

| 템플릿 | 용도 | 자동 생성 필드 |
|--------|------|----------------|
| **NOTE** | 일반 노트 | 제목, 태그, 생성일 |
| **SKETCH** | 시각적 다이어그램 | 제목, 캔버스 데이터 |
| **MTG** | 회의록 | 제목, 참석자, 안건, 날짜 |
| **SEM** | 세미나 노트 | 제목, 발표자, 주제 |
| **EVENT** | 이벤트 기록 | 제목, 날짜, 장소 |
| **OFA** | 공무 사항 | 제목, 카테고리, 상태 |
| **PAPER** | 연구 논문 | 제목, 저자, DOI, 초록 |
| **LIT** | 문헌 노트 | 제목, 출처, 핵심 논점 |
| **DATA** | 데이터 문서 | 제목, 출처, 방법론 |
| **THEO** | 이론 탐구 | 제목, 도메인, 전제 |
| **CONTACT** | 연락처 카드 | 이름, 소속, 이메일, 전화 |
| **SETUP** | 설정 노트 | 제목, 카테고리 |

**각 템플릿 제공 사항:**
- 날짜 접두사 자동 파일명 (예: `MTG_260208_주간 스탠드업.md`)

### 9. 휴대 가능한 보관함 &mdash; 노트를 어디서든 사용

보관함은 그냥 폴더입니다. 데이터베이스도, 독점 형식도 없습니다:

- **NAS** &mdash; Synology, QNAP 등 NAS에 보관함을 두고 어떤 컴퓨터에서든 접근
- **외장 드라이브** &mdash; USB 드라이브나 외장 SSD에 넣고 휴대
- **클라우드 동기화** &mdash; Dropbox, Google Drive, OneDrive 등 아무 동기화 서비스와 호환

**공유 저장소를 위한 안전 장치:**

| 기능 | 설명 |
|------|------|
| **보관함 잠금** | 두 기기가 동시에 같은 보관함을 편집하는 것을 방지 |
| **충돌 감지** | 동기화 충돌을 감지하고 수동 해결을 안내 |
| **원자적 쓰기** | 임시 파일 + 이름 변경 패턴으로 동기화 중 파일 손상 방지 |
| **벌크 동기화 인식** | 대량 동기화 시 UI 업데이트를 일시 중지하여 깜빡임 방지 |

### 녹화 가이드라인

- **테마**: 기본적으로 다크 테마 사용; 히어로 이미지와 그래프 뷰 GIF에는 라이트 테마 변형 포함
- **콘텐츠**: 현실적인 샘플 데이터 사용 ("Lorem ipsum" 금지). 여러 템플릿으로 20-30개 노트가 있는 데모 보관함 생성
- **커서**: 자연스러운 마우스 움직임. 클릭 전 주요 UI 요소 위에서 잠시 멈춤
- **프레임 레이트**: GIF는 15-20 FPS (좋은 품질, 적절한 파일 크기)
- **형식**: 정적 이미지는 PNG, 애니메이션은 GIF. GIF는 5 MB 이하로 최적화
- **도구**: [ScreenToGif](https://www.screentogif.com/) (무료, 오픈소스) 또는 [ShareX](https://getsharex.com/) 추천
