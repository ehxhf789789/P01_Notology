## Notology란?

Notology는 구조화된 노트 템플릿, 위키링크 연결, Tantivy 전문 검색 엔진을 하나의 네이티브 앱에 결합한 **데스크톱 지식 관리 앱**입니다. 모든 노트는 플레인 마크다운(`.md`) 파일로 저장되며, 보관함은 그냥 폴더이기 때문에 USB, NAS, 클라우드 동기화 폴더 어디에 넣어도 작동합니다.

**핵심 원칙:**

| 원칙 | 설명 |
|------|------|
| **플레인 마크다운** | 독점 형식 없음, 종속성 없음 &mdash; `.md` 파일은 어떤 텍스트 에디터로든 열 수 있습니다 |
| **휴대 가능한 보관함** | 보관함은 그냥 폴더입니다. 복사, 이동, 동기화 자유 |
| **타입이 있는 노트** | 12가지 템플릿 (회의, 논문, 연락처 등)으로 빈 페이지가 아닌 구조화된 노트 작성 |
| **네이티브 성능** | Rust 백엔드 (Tauri v2) + React 프론트엔드. Electron 오버헤드 없음 |
| **오프라인 우선** | 계정 없음, 구독 없음, 인터넷 불필요 |
| **이중 언어** | 한국어 / English 완전 지원 (UI, 템플릿, 본문 안내 모두) |

## 빠른 시작

```
1. Releases 페이지에서 다운로드 & 설치
2. Notology 실행
3. "보관함 열기" 클릭 → 아무 폴더 선택 (빈 폴더 또는 기존 마크다운 폴더)
4. Ctrl+N → 12가지 템플릿 중 선택 → 작성 시작
5. [[ 를 입력하여 다른 노트 연결
6. Ctrl+Shift+F → 그래프 탭으로 지식 네트워크 시각화
```

> **팁:** 보관함 폴더를 Synology NAS나 외장 드라이브에 두면 여러 컴퓨터에서 동일한 보관함을 사용할 수 있습니다. 보관함 잠금 시스템이 동시 편집을 안전하게 방지합니다.

## 핵심 개념

Notology를 효과적으로 사용하기 위해 알아야 할 핵심 개념입니다. 각 개념은 서로 연결되어 하나의 체계적인 지식 관리 워크플로우를 형성합니다.

### 보관함 (Vault)

보관함은 Notology가 관리하는 **최상위 폴더**입니다. 보관함 안의 모든 `.md` 파일은 자동으로 인덱싱되고, 위키링크로 연결되며, 그래프에 표시됩니다.

```
내 보관함/                    ← 보관함 루트 (이 폴더를 선택)
├── .notology/               ← Notology 메타데이터 (자동 생성)
│   ├── vault.lock           ← 보관함 잠금 파일
│   ├── tag-ontology.yaml    ← 태그 온톨로지 정의
│   ├── backups/             ← 자동 백업 (파일당 최대 5개)
│   └── templates/           ← 커스텀 템플릿
├── 프로젝트/
│   ├── 프로젝트.md          ← 폴더 노트 (폴더명과 동일)
│   ├── NOTE_아이디어.md
│   └── MTG_260208_킥오프.md
├── 연구/
│   ├── 연구.md
│   ├── PAPER_Attention.md
│   └── LIT_딥러닝 개론.md
└── 연락처/
    ├── CONTACT_김철수.md
    └── CONTACT_이영희.md
```

**보관함의 특징:**
- 보관함은 일반 폴더이므로, 파일 탐색기에서 직접 파일을 추가/삭제/이동할 수 있습니다
- `.notology/` 디렉토리를 제외하면 모든 파일이 표준 마크다운이므로 다른 앱에서도 열 수 있습니다
- 여러 보관함을 만들어 업무/개인/학습을 분리할 수 있습니다
- 보관함을 NAS, 외장 드라이브, 클라우드 동기화 폴더에 넣어 여러 기기에서 접근할 수 있습니다

### 폴더 노트 (Folder Note)

폴더 노트는 **폴더명과 동일한 이름의 마크다운 파일**입니다. 해당 폴더의 "표지" 역할을 하며, 폴더의 목적, 하위 항목의 개요, 링크 모음 등을 담습니다.

```
프로젝트/
├── 프로젝트.md       ← 이것이 폴더 노트 (폴더명 "프로젝트"과 동일)
├── NOTE_요구사항.md
└── MTG_260210_정기회의.md
```

- 사이드바에서 폴더를 생성하면 폴더 노트가 **자동으로** 함께 생성됩니다
- 폴더 노트의 타입은 `CONTAINER`이며, 깊이(L0, L1, L2+)에 따라 다른 CSS 클래스가 적용됩니다
- 그래프 뷰에서 폴더 노트는 별도 색상으로 강조되어 구조를 한눈에 파악할 수 있습니다

### 프론트매터 & YAML

모든 노트는 파일 상단에 YAML 형식의 **프론트매터**를 포함합니다. 프론트매터는 노트의 메타데이터를 구조화하여 저장하며, 검색, 필터링, 그래프 연결, 상태 추적에 활용됩니다.

```yaml
---
id: "20260209143022"
title: "분기 리뷰 회의"
type: MTG
created: "2026-02-09T14:30:22.000Z"
modified: "2026-02-09T15:45:10.000Z"
state:
  workflow: in-progress
  confidence: verified
  maturity: 3
tags:
  domain:
    - domain/프로젝트-관리
  who:
    - who/김철수
    - who/이영희
  org:
    - org/개발팀
  ctx:
    - ctx/2026-Q1
  source: []
  method: []
  status:
    - status/진행중
relations:
  - relation_type: extends
    target: "NOTE_프로젝트 개요"
    strength: 0.8
cssclasses:
  - mtg-type
participants:
  - 김철수
  - 이영희
  - 박민수
date: "2026-02-09T14:30:22.000Z"
---
```

프론트매터는 Notology가 자동으로 생성하고 관리하지만, 사용자가 직접 편집할 수도 있습니다. 자세한 스키마는 아래 [프론트매터 레퍼런스](#프론트매터-레퍼런스) 섹션을 참고하세요.

### 패싯 태그 (Faceted Tags)

Notology의 태그 시스템은 단순한 `#태그`가 아닌, **7개 네임스페이스로 분류된 패싯(faceted) 태그**를 사용합니다. 이를 통해 노트를 다양한 관점에서 분류하고, 교차 검색할 수 있습니다.

| 네임스페이스 | 접두사 | 용도 | 예시 |
|------------|--------|------|------|
| **domain** | `domain/` | 주제, 분야, 학문 영역 | `domain/머신러닝`, `domain/프로젝트-관리` |
| **who** | `who/` | 관련 인물 | `who/김철수`, `who/지도교수` |
| **org** | `org/` | 조직, 팀, 기관 | `org/개발팀`, `org/서울대학교` |
| **ctx** | `ctx/` | 맥락, 시기, 상황 | `ctx/2026-Q1`, `ctx/출장` |
| **source** | `source/` | 출처, 참고 자료 | `source/논문`, `source/강의` |
| **method** | `method/` | 방법론, 도구, 프레임워크 | `method/실험`, `method/인터뷰` |
| **status** | `status/` | 프로젝트/작업 상태 | `status/진행중`, `status/완료` |

태그는 `.notology/tag-ontology.yaml` 파일에 온톨로지(정의, 동의어, 스타일)로 저장되어 보관함과 함께 휴대됩니다. 자세한 내용은 [태그 시스템](#태그-시스템) 섹션을 참고하세요.

### 위키링크 (Wiki-link)

`[[이중 괄호]]`로 보관함 내 다른 노트를 참조합니다. 위키링크는 지식 그래프의 연결선이 되며, 양방향 추적이 가능합니다.

- `[[노트 제목]]` &mdash; 다른 노트를 참조
- `![[이미지.png]]` &mdash; 이미지를 인라인으로 임베드
- `[[` 를 입력하면 보관함 내 모든 노트에서 실시간 자동완성
- 노트 이름을 변경하면 해당 노트를 참조하는 **모든 위키링크가 자동으로 갱신**됩니다

### 호버 윈도우 (Hover Window)

호버 윈도우는 앱 내부에서 떠다니는 **플로팅 윈도우**입니다. 메인 에디터를 떠나지 않고 여러 문서를 동시에 열어 참조하거나 편집할 수 있습니다.

- 5가지 콘텐츠 타입 지원: 에디터(`.md`), PDF, 이미지, 코드, 웹(URL)
- 드래그로 자유 배치, 모서리 드래그로 리사이즈, 화면 가장자리 스냅
- 최소화하면 하단 작업표시줄 스타일로 정리되며, 클릭으로 복원
- 콘텐츠 캐싱으로 창 간 전환이 즉각적

### 노트 상태 (Note State)

모든 노트는 3가지 차원의 **상태**를 가집니다. 이를 통해 노트의 완성도와 신뢰도를 체계적으로 추적할 수 있습니다.

| 차원 | 설명 | 가능한 값 |
|------|------|----------|
| **workflow** | 작업 진행 단계 | `draft` (초안), `in-progress` (진행중), `review` (검토), `final` (완료), `archived` (보관) |
| **confidence** | 정보 신뢰도 | `unverified` (미검증), `verified` (검증됨), `outdated` (오래됨), `disputed` (논쟁중) |
| **maturity** | 내용 성숙도 | 1~5 스케일 (1: 메모 수준, 5: 완성된 문서) |

```yaml
state:
  workflow: in-progress
  confidence: verified
  maturity: 3
```

### 작가 & 스토리텔러

<!-- 이미지: docs/images/usecase-writer.png -->
<!-- ![작가 활용 사례](docs/images/usecase-writer.png) -->

소설의 세계관을 직관적으로 관리하세요. **CONTACT 템플릿**을 캐릭터 시트로 활용하여 성격, 배경, 관계를 구조화된 필드로 정리할 수 있습니다. **폴더 노트**로 장소, 플롯 아크, 타임라인을 구성하세요. **그래프 뷰**는 캐릭터, 사건, 배경 사이의 숨겨진 연결을 드러내어 스토리 세계의 전체 조감도를 제공합니다.

> *예시 보관함 구조:*
> ```
> 나의 소설/
> ├── 등장인물/
> │   ├── 등장인물.md              (폴더 노트: 캐릭터 인덱스)
> │   ├── CONTACT_아리아.md        (캐릭터 시트)
> │   └── CONTACT_마르쿠스.md
> ├── 장소/
> │   ├── 장소.md
> │   └── NOTE_오래된 도서관.md
> ├── 플롯 아크/
> │   ├── NOTE_1막 - 부름.md
> │   └── NOTE_2막 - 여정.md
> └── 자료 조사/
>     ├── LIT_중세 건축.md
>     └── DATA_역사 연표.md
> ```

### 연구자 & 학자

<!-- 이미지: docs/images/usecase-researcher.png -->
<!-- ![연구자 활용 사례](docs/images/usecase-researcher.png) -->

개인 연구 라이브러리를 구축하세요. **PAPER 템플릿**으로 읽은 논문마다 DOI, 저자, 초록을 체계적으로 기록합니다. `[[위키링크]]`로 논문 간 아이디어의 계보를 추적하세요. **호버 윈도우**로 PDF를 읽으면서 동시에 노트를 작성할 수 있습니다. **THEORY 템플릿**으로 가설을 탐구하고, **DATA 템플릿**으로 데이터셋과 방법론을 문서화하세요.

- **문헌 리뷰**: LIT 템플릿으로 출처, 핵심 논점, 비판적 노트를 기록
- **교차 참조**: 논문, 이론, 데이터 간 위키링크가 상호 연결된 지식 베이스를 형성
- **즉각적인 검색**: Tantivy 전문 검색으로 수천 개의 노트에서 논문, 인용문, 개념을 밀리초 내에 검색

### 직장인 & 사무직

<!-- 이미지: docs/images/usecase-office.png -->
<!-- ![직장인 활용 사례](docs/images/usecase-office.png) -->

일상 업무와 개인 지식 관리를 효율화하세요. **MTG 템플릿**이 참석자, 안건, 액션 아이템이 포함된 회의록을 자동 생성합니다. **캘린더 뷰**로 마감일을 추적하세요. 폴더 노트로 프로젝트를 정리하고 **5가지 검색 모드**로 무엇이든 즉시 찾을 수 있습니다. **메모 시스템**으로 원본을 수정하지 않고 문서에 주석을 달 수 있습니다.

- **회의록 작성**: MTG 템플릿의 날짜 접두사 파일명으로 시간순 정리
- **프로젝트 정리**: 프로젝트별 폴더, 폴더 노트를 프로젝트 대시보드로 활용
- **휴대 가능한 업무 환경**: NAS에 보관함을 두고 사무실, 집, 이동 중 어디서든 접근

### 학생 (고등학생 & 대학생)

<!-- 이미지: docs/images/usecase-student.png -->
<!-- ![학생 활용 사례](docs/images/usecase-student.png) -->

모든 과목의 학습을 체계화하세요. **SEM 템플릿**으로 발표자와 주제 필드가 포함된 강의 노트를 작성합니다. **위키링크**로 과목을 넘나드는 개념 맵을 만드세요 &mdash; 물리 개념을 수학 유도과정에, 수학을 실험 보고서에 연결합니다. **그래프 뷰**는 지식이 어떻게 연결되는지 보여주어, 시각적 연상을 통해 기억을 강화합니다.

- **학습 노트**: 콜아웃 블록, 코드 블록, 수식 표기가 가능한 마크다운 서식
- **시험 준비**: 전체 노트 대상 전문 검색 &mdash; 특정 개념이 언급된 모든 곳을 즉시 검색
- **지식 매핑**: 그래프 뷰로 어떤 주제가 잘 연결되어 있고 어떤 주제를 더 공부해야 하는지 파악
- **휴대성**: USB 드라이브 하나에 전체 학습 보관함을 넣어 집, 학교, 도서관을 오가며 사용

## Notology vs Obsidian

Notology는 Obsidian의 보관함 기반 접근과 위키링크 철학에서 영감을 받았습니다. 차이점은 다음과 같습니다:

| | **Notology** | **Obsidian** |
|--|-------------|-------------|
| **엔진** | Tauri v2 (Rust + WebView) | Electron (Chromium) |
| **검색** | Tantivy 전문 검색 엔진 (Rust) | 내장 파일 검색 |
| **노트 타입** | 12개 구조화된 템플릿 + 자동 프론트매터 | 빈 마크다운 + 커뮤니티 템플릿 |
| **태그 시스템** | 7개 네임스페이스 패싯 태그 + 온톨로지 | 플랫한 `#태그` |
| **노트 상태** | 3차원 상태 (workflow / confidence / maturity) | 없음 (플러그인 필요) |
| **멀티 윈도우** | 내장 호버 윈도우 (드래그, 스냅, 리사이즈) | 팝아웃 윈도우 (별도 OS 창) |
| **캔버스** | 플로우차트 도형 (다이아몬드, 평행사변형, 원) + 화살표 | 카드와 연결선 |
| **코멘트** | 내장 노트별 코멘트/메모 시스템 | 플러그인 필요 |
| **그래프** | 물리 제어 + 타입별 컬러링 | 내장 그래프 |
| **휴대성** | 보관함 잠금 + Synology 충돌 감지 + 원자적 쓰기 | 유료 Sync 서비스 |
| **가격** | 무료 오픈소스 | 코어 무료, Sync/Publish 유료 |
| **코드** | 오픈소스 (MIT) | 비공개 |

## 기능 가이드

### 1. 리치 텍스트 에디터

TipTap 기반 에디터로 마크다운을 실시간 리치 텍스트로 렌더링합니다.

<!-- GIF: docs/gifs/01-editor.gif -->
<!-- ![에디터](docs/gifs/01-editor.gif) -->

**기본 서식:**
- **마크다운 + WYSIWYG 하이브리드** &mdash; `# `, `- `, `> ` 입력 시 즉시 렌더링
- **제목** &mdash; H1~H6 (Ctrl+1 ~ Ctrl+6)
- **텍스트 서식** &mdash; 굵게, 기울임, 밑줄, 취소선, 하이라이트, 인라인 코드
- **위첨자 / 아래첨자** &mdash; 과학 표기법과 화학식에 유용

**블록 요소:**
- **6종 콜아웃 블록** &mdash; 정보(파란), 경고(노란), 오류(빨간), 성공(초록), 노트(보라), 팁(청록). 각각 컬러 테두리로 시각적으로 구분
- **테이블** &mdash; 색상 지원 셀 + 헤더 행
- **코드 블록** &mdash; highlight.js 기반 구문 강조 (180+ 언어)
- **할일 목록** &mdash; 인터랙티브 체크박스 (클릭으로 완료/미완료 전환)
- **인용문** &mdash; `>` 블록쿼트

**편의 기능:**
- **접을 수 있는 툴바** &mdash; 전체 서식 바 또는 미니멀 모드 전환
- **커스텀 키보드 단축키** &mdash; 모든 동작 재매핑 가능 (설정 > 단축키)
- **자동 저장** &mdash; 편집 내용이 자동으로 저장

> **팁:** 콜아웃 블록은 학습 노트에서 핵심 내용을 강조하거나, 회의록에서 결정 사항을 표시할 때 유용합니다. `정보` 타입으로 배경 지식을, `경고` 타입으로 주의사항을, `팁` 타입으로 실행 아이템을 표시해 보세요.

### 2. 위키링크 & 백링크

지식 그래프의 근간입니다. `[[이중 괄호]]`로 노트를 자유롭게 연결하세요.

<!-- GIF: docs/gifs/02-wikilinks.gif -->
<!-- ![위키링크](docs/gifs/02-wikilinks.gif) -->

- **자동 완성** &mdash; `[[` 입력 시 보관함 내 전체 노트에서 실시간 추천 (제목 기반)
- **이미지 임베딩** &mdash; `![[photo.png]]`으로 인라인 이미지 렌더링
- **이름 변경 시 자동 업데이트** &mdash; 노트 이름을 바꾸면 보관함 전체에서 해당 노트를 참조하는 모든 `[[링크]]`가 자동 갱신
- **백링크 추적** &mdash; 검색 상세 뷰에서 현재 노트를 참조하는 모든 노트 확인
- **폴더 간 링크** &mdash; 서로 다른 폴더의 노트도 자유롭게 연결 (경로 지정 불필요)

> **팁:** 노트를 작성하면서 떠오르는 개념에 `[[]]`를 씌우세요. 아직 그 노트가 존재하지 않아도 괜찮습니다 &mdash; 나중에 해당 제목의 노트를 만들면 자동으로 연결됩니다.

### 3. 12가지 구조화된 노트 템플릿

빈 페이지 노트 앱과 달리, Notology는 구조화된 프론트매터가 포함된 **타입이 있는 노트**를 제공합니다. Ctrl+N으로 템플릿 선택 모달을 열고, 12가지 중 하나를 선택하면 프론트매터와 본문 구조가 자동 생성됩니다.

<!-- GIF: docs/gifs/03-templates.gif -->
<!-- ![템플릿](docs/gifs/03-templates.gif) -->

| 템플릿 | 접두사 | 용도 | 고유 프론트매터 필드 | 자동 생성 본문 섹션 |
|--------|--------|------|---------------------|-------------------|
| **NOTE** | `NOTE` | 일반 노트 | &mdash; | 개요, 내용 |
| **SKETCH** | `SKETCH` | 시각적 다이어그램 (캔버스) | `canvas: true` | (캔버스 모드) |
| **MTG** | `MTG` | 회의록 | `participants`, `date` | 회의 정보, 안건, 논의, 결정 |
| **SEM** | `SEM` | 세미나/강의 노트 | &mdash; | 개요, 내용, 핵심 포인트 |
| **EVENT** | `EVENT` | 이벤트 기록 | `date`, `location`, `organizer`, `participants` | 이벤트 정보, 개요, 일정 |
| **OFA** | `OFA` | 공무/행정 사항 | &mdash; | 개요, 내용, 결정 |
| **PAPER** | `PAPER` | 연구 논문 분석 | `authors`, `venue`, `year`, `doi`, `url` | 논문 정보, 요약, 기여, 방법론, 결과 |
| **LIT** | `LIT` | 문헌 노트/독서록 | `authors`, `year`, `publisher`, `source`, `url` | 문헌 정보, 요약, 핵심 내용 |
| **DATA** | `DATA` | 데이터셋 문서화 | &mdash; | 개요, 데이터 설명 |
| **THEO** | `THEO` | 이론/개념 탐구 | &mdash; | 정의, 배경, 핵심 개념, 응용, 관련 이론 |
| **CONTACT** | `CONTACT` | 연락처/인물 카드 | `email`, `phone`, `organization`, `role` | 연락처 정보, 이력, 관련 노트 |
| **SETUP** | `SETUP` | 설정/환경 구성 메모 | &mdash; | 개요, 맥락 |

**모든 템플릿 공통 제공 사항:**
- 14자리 타임스탬프 기반 고유 ID (예: `20260209143022`)
- ISO 8601 형식의 `created` / `modified` 타임스탬프
- 3차원 상태 (`workflow`, `confidence`, `maturity`)
- 7개 네임스페이스 패싯 태그
- `relations` 배열 (노트 간 의미적 관계)
- `cssclasses` (타입별 자동 스타일링)
- 날짜 접두사 자동 파일명 (예: `MTG_260208_주간_스탠드업.md`)

**템플릿 커스터마이징:**
설정 > 템플릿 편집기에서 각 템플릿의 활성화/비활성화, 기본 태그 카테고리 설정, 필드 커스터마이즈가 가능합니다.

### 4. 호버 윈도우 (멀티 윈도우 편집)

앱을 떠나지 않고 여러 노트를 플로팅 윈도우로 동시에 열 수 있습니다.

<!-- GIF: docs/gifs/04-hover-windows.gif -->
<!-- ![호버 윈도우](docs/gifs/04-hover-windows.gif) -->

**배치와 크기 조절:**
- **드래그 & 드롭** &mdash; 제목 바를 잡고 화면 어디든 자유롭게 배치
- **리사이즈** &mdash; 모서리 또는 변 드래그로 크기 조절
- **스냅 존** &mdash; 화면 가장자리로 드래그하면 반 화면, 1/4 화면 등 자동 정렬
- **4가지 크기 프리셋** &mdash; S, M, L, XL 키보드 단축키로 빠른 크기 변경

**윈도우 관리:**
- **최소화 / 복원** &mdash; Windows 작업표시줄 스타일. 최소화된 윈도우는 하단에 탭으로 표시되며, 클릭으로 복원
- **포커스 관리** &mdash; 여러 윈도우가 열려 있을 때 클릭한 윈도우가 최상위로 올라옴
- **줌** &mdash; Ctrl+스크롤로 윈도우별 독립 확대/축소
- **콘텐츠 캐싱** &mdash; 한번 열린 콘텐츠는 캐시되어 재접근 시 즉시 표시

**5가지 콘텐츠 타입:**

| 타입 | 열리는 조건 | 기능 |
|------|------------|------|
| **에디터** | `.md` 파일 | 툴바 포함 전체 편집, 프론트매터 수정 가능 |
| **PDF** | `.pdf` 파일 | 내장 PDF 뷰어, 페이지 이동 |
| **이미지** | `.png`, `.jpg`, `.gif` 등 | 줌 가능 미리보기 |
| **코드** | `.js`, `.py`, `.rs` 등 | 구문 강조 읽기 전용 뷰 |
| **웹** | URL | 내장 웹 미리보기 |

> **팁:** 논문을 PDF 호버 윈도우로 열고, 옆에 PAPER 템플릿 노트를 에디터 호버 윈도우로 열면, 논문을 읽으면서 동시에 메모를 작성하는 이상적인 연구 환경을 만들 수 있습니다.

### 5. 인터랙티브 그래프 뷰

전체 지식 네트워크를 힘-방향(force-directed) 그래프로 시각화합니다.

<!-- GIF: docs/gifs/05-graph.gif -->
<!-- ![그래프 뷰](docs/gifs/05-graph.gif) -->

**노드 타입과 시각화:**
- **노트 노드** &mdash; 템플릿 타입별 고유 컬러로 표시 (NOTE, MTG, PAPER 등 각각 다른 색상)
- **태그 노드** &mdash; 네임스페이스별 별도 표시
- **첨부파일 노드** &mdash; PDF, 이미지 등 첨부파일
- **폴더 노트** &mdash; 별도 테두리/색상으로 강조

**물리 시뮬레이션 설정:**
d3-force 기반으로 4가지 물리 파라미터를 슬라이더로 조절할 수 있습니다:

| 파라미터 | 설명 | 효과 |
|---------|------|------|
| **척력 (Repulsion)** | 노드 간 반발력 | 값을 높이면 노드가 넓게 퍼짐 |
| **링크 거리 (Link Distance)** | 연결된 노드 간 기본 거리 | 값을 높이면 연결선이 길어짐 |
| **중력 (Gravity)** | 중심을 향한 끌어당김 | 값을 높이면 그래프가 중앙에 모임 |
| **중심력 (Center Force)** | 전체 그래프의 중심 정렬 | 값을 높이면 화면 중심에 고정 |

**필터와 탐색:**
- 노트 타입, 태그, 검색어로 필터링 가능 &mdash; 비매칭 노드는 희미해지고, 매칭 노드가 강조
- 노드 클릭으로 호버 윈도우로 바로 열기
- 실시간 반영 &mdash; 노트 생성/편집 시 그래프 자동 업데이트
- **다크 & 라이트 모드** &mdash; 테마에 따라 완전 적응

**설정 저장:**
물리 시뮬레이션 설정은 보관함별로 영구 저장되므로, 한번 조절하면 다음 실행 시에도 유지됩니다.

### 6. 캔버스 에디터

플로우차트, 마인드맵, 다이어그램을 만드는 공간적 사고 도구입니다. SKETCH 템플릿을 선택하면 캔버스 모드로 진입합니다.

<!-- GIF: docs/gifs/06-canvas.gif -->
<!-- ![캔버스](docs/gifs/06-canvas.gif) -->

- **무한 캔버스** &mdash; 드래그로 팬, 스크롤로 줌 인/아웃
- **4종 도형:**

| 도형 | 용도 | 설명 |
|------|------|------|
| **직사각형** | 프로세스/작업 | 일반적인 프로세스 단계 |
| **다이아몬드** | 분기/결정 | 예/아니오 조건 분기 |
| **원** | 시작/종료 | 프로세스의 시작점과 끝점 |
| **평행사변형** | 입출력 | 데이터 입력/출력 |

- **연결 화살표** &mdash; 도형 간 화살표를 그려 흐름 표시
- **노드 내 리치 텍스트** &mdash; 단순 텍스트가 아닌 서식 지원 (굵게, 기울임 등)
- **프론트매터에 저장** &mdash; 캔버스 데이터가 노트 파일 내 프론트매터에 저장되어, 별도 캔버스 파일이 필요 없음

### 7. 전문 검색 (5가지 모드)

**Tantivy** (Apache Lucene의 Rust 구현체) 기반으로, 수천 개의 노트에서도 밀리초 내에 검색 결과를 제공합니다. Ctrl+Shift+F로 검색 패널을 열 수 있습니다.

<!-- GIF: docs/gifs/07-search.gif -->
<!-- ![검색](docs/gifs/07-search.gif) -->

| 모드 | 기능 | 예시 | 활용 팁 |
|------|------|------|---------|
| **노트** | 제목, 태그, 노트 타입 검색 | `type:MTG tag:project-alpha` | 특정 타입의 노트만 빠르게 필터링 |
| **본문** | 전문 콘텐츠 검색 + 매칭 부분 하이라이팅 | `"분기 리뷰"` | 본문 속 특정 문구 찾기 |
| **첨부파일** | 파일명, 확장자, 크기로 검색 | `*.pdf` | 보관함 내 모든 첨부파일 탐색 |
| **상세** | 메타데이터 브라우저 + 다중 필터 | 날짜 범위 + 타입 + 태그 조합 | 복합 조건 검색, 메모 수 확인 |
| **그래프** | 검색 결과가 하이라이팅된 시각적 그래프 | 노드 클릭으로 이동 | 검색 결과의 연결 관계 시각화 |

**검색 인덱스:**
- 보관함을 열면 Tantivy가 모든 `.md` 파일을 자동 인덱싱
- 파일 변경 시 `notify` 파일 감시기가 200ms 디바운스로 인덱스를 실시간 갱신
- 프론트매터 필드(제목, 타입, 태그)와 본문 모두 인덱싱

### 8. 캘린더 뷰

할일, 메모, 이벤트를 월간 캘린더에서 추적합니다. Ctrl+Shift+C로 열 수 있습니다.

<!-- GIF: docs/gifs/08-calendar.gif -->
<!-- ![캘린더](docs/gifs/08-calendar.gif) -->

- **월간 그리드** &mdash; 각 날짜에 할일 수가 점(dot)으로 표시
- **날짜 상세** &mdash; 날짜를 클릭하면 해당일의 모든 노트, 할일 체크박스, 메모가 상세 패널에 표시
- **노트 생성** &mdash; 캘린더에서 직접 날짜를 선택하고 "노트 만들기"로 해당일 노트 생성
- **월 이동** &mdash; 화살표 버튼으로 이전/다음 달 탐색

> **팁:** MTG 템플릿이나 EVENT 템플릿으로 생성한 노트는 `date` 프론트매터가 자동 설정되므로 캘린더에 자동으로 표시됩니다.

### 9. 코멘트 & 메모 시스템

노트 본문을 수정하지 않고 영구적인 주석을 추가할 수 있습니다. Ctrl+M으로 메모 패널을 토글합니다.

<!-- 이미지: docs/images/10-comments.png -->
<!-- ![코멘트](docs/images/10-comments.png) -->

- **인라인 코멘트** &mdash; 텍스트를 드래그 선택 후 코멘트 추가. 해당 텍스트가 하이라이트되고 말풍선이 표시
- **독립 메모** &mdash; 노트 전체에 대한 독립적인 메모 작성. 파일에 첨부되지만 본문에 영향 없음
- **할일 추적** &mdash; 코멘트에 체크박스를 포함하여 액션 아이템 관리
- **메모 수 뱃지** &mdash; 검색 결과 목록에서 각 노트의 메모 수가 뱃지로 표시되어 빠른 개요 제공

> **팁:** 리뷰 워크플로우에서 유용합니다. 동료가 작성한 노트의 본문을 수정하지 않고, 코멘트로 피드백을 남겨보세요.

## 프론트매터 레퍼런스

모든 노트의 YAML 프론트매터 스키마입니다. 아래는 전체 필드 정의이며, 모든 필드는 자동 생성되지만 수동 편집도 가능합니다.

### 기본 필드 (모든 노트 타입 공통)

```yaml
---
# === 식별 ===
id: "20260209143022"          # 14자리 타임스탬프 (YYYYMMDDHHMMSS), 자동 생성
title: "노트 제목"             # 노트 제목
type: NOTE                    # 노트 타입: NOTE | SKETCH | MTG | SEM | EVENT | OFA |
                              #            PAPER | LIT | DATA | THEO | CONTACT | SETUP |
                              #            CONTAINER

# === 시간 ===
created: "2026-02-09T14:30:22.000Z"    # 생성 시각 (ISO 8601)
modified: "2026-02-09T15:45:10.000Z"   # 최종 수정 시각 (자동 갱신)

# === 상태 ===
state:
  workflow: draft              # draft | in-progress | review | final | archived
  confidence: unverified       # unverified | verified | outdated | disputed
  maturity: 1                  # 1-5 (1: 메모, 5: 완성 문서)

# === 태그 ===
tags:
  domain: []                   # 주제/분야 (예: domain/머신러닝)
  who: []                      # 관련 인물 (예: who/김철수)
  org: []                      # 조직/기관 (예: org/개발팀)
  ctx: []                      # 맥락/시기 (예: ctx/2026-Q1)
  source: []                   # 출처 (예: source/논문)
  method: []                   # 방법론 (예: method/실험)
  status: []                   # 프로젝트 상태 (예: status/진행중)

# === 관계 ===
relations:                     # 다른 노트와의 의미적 관계
  - relation_type: extends     # supports | refutes | extends | implements |
                               # derives-from | part-of | is-example-of | causes
    target: "대상 노트 제목"
    strength: 0.8              # 관계 강도 (0.0 ~ 1.0, 선택)

# === 스타일 ===
cssclasses:                    # UI 스타일링용 CSS 클래스 (자동)
  - note-type                  # note-type | mtg-type | paper-type | theo-type |
                               # contact-type | container-type | lit-type |
                               # event-type | ofa-type | sem-type | data-type |
                               # setup-type | sketch-type
---
```

### 타입별 추가 필드

**MTG (회의)**
```yaml
participants:                  # 참석자 목록
  - 김철수
  - 이영희
date: "2026-02-09T14:30:22.000Z"   # 회의 일시
```

**PAPER (연구 논문)**
```yaml
authors:                       # 저자 목록
  - "A. Smith"
  - "B. Jones"
venue: "NeurIPS 2025"          # 학회/저널명
year: 2025                     # 출판 연도
doi: "10.1234/example"         # DOI 식별자
url: "https://arxiv.org/..."   # 논문 URL
```

**LIT (문헌)**
```yaml

  - "홍길동"
year: 2024                     # 출판 연도
publisher: "출판사명"           # 출판사
source: "문헌 출처"             # 출처 정보
url: "https://..."             # URL
```

**EVENT (이벤트)**

```yaml
  - 참가자1
  - 참가자2
```

**CONTACT (연락처)**

```yaml
  - "@홍길동"
```


### Relation 타입 레퍼런스

| 관계 타입 | 의미 | 예시 |
|----------|------|------|
| `supports` | A가 B를 뒷받침 | 실험 결과가 가설을 지지 |
| `refutes` | A가 B를 반박 | 새 데이터가 기존 이론을 반박 |
| `extends` | A가 B를 확장 | 후속 연구가 기존 연구를 확장 |
| `implements` | A가 B를 구현 | 코드가 알고리즘을 구현 |
| `derives-from` | A가 B에서 파생 | 아이디어가 논문에서 파생 |
| `part-of` | A가 B의 일부 | 챕터가 책의 일부 |
| `is-example-of` | A가 B의 사례 | 사례 연구가 이론의 사례 |
| `causes` | A가 B를 유발 | 원인이 결과를 유발 |

## 태그 시스템

Notology의 태그 시스템은 **패싯 분류(faceted classification)** 방식을 사용합니다. 하나의 노트에 여러 관점의 태그를 붙여 다차원 검색과 필터링이 가능합니다.

### 7개 태그 네임스페이스

#### domain (주제)
노트의 주제, 분야, 학문 영역을 나타냅니다.
```yaml
tags:
  domain:
    - domain/머신러닝
    - domain/자연어처리
```
**활용 예시:** `domain/물리학`, `domain/경제학`, `domain/UI-디자인`, `domain/프로젝트-관리`

#### who (인물)
노트에 관련된 사람을 태그합니다. CONTACT 노트와 연결하여 인물 중심 탐색이 가능합니다.
```yaml
tags:
  who:
    - who/김철수
    - who/지도교수
```
**활용 예시:** `who/멘토`, `who/공동저자`, `who/클라이언트-A`

#### org (조직)
관련 조직, 팀, 기관, 회사를 표시합니다.
```yaml
tags:
  org:
    - org/개발팀
    - org/서울대학교
```
**활용 예시:** `org/인사팀`, `org/연구실`, `org/스타트업-X`

#### ctx (맥락)
시간적 맥락, 상황, 프로젝트 단계를 나타냅니다.
```yaml
tags:
  ctx:
    - ctx/2026-Q1
    - ctx/출장-도쿄
```
**활용 예시:** `ctx/스프린트-3`, `ctx/학기-2026-1`, `ctx/온보딩`

#### source (출처)
정보의 출처나 획득 경로를 표시합니다.
```yaml
tags:
  source:
    - source/논문
    - source/강의노트
```
**활용 예시:** `source/인터뷰`, `source/웹`, `source/책`, `source/컨퍼런스`

#### method (방법론)
사용된 연구 방법론, 도구, 프레임워크를 나타냅니다.
```yaml
tags:
  method:
    - method/실험
    - method/정성분석
```
**활용 예시:** `method/설문조사`, `method/A-B-테스트`, `method/문헌리뷰`, `method/프로토타이핑`

#### status (프로젝트 상태)
프로젝트나 작업의 현재 진행 상태를 표시합니다. `state.workflow`와 달리 프로젝트/이슈 수준의 상태입니다.
```yaml
tags:
  status:
    - status/진행중
```
**활용 예시:** `status/대기`, `status/완료`, `status/보류`, `status/긴급`

### 태그 온톨로지

모든 태그 정의는 `.notology/tag-ontology.yaml` 파일에 저장됩니다. 이 파일은 보관함과 함께 이동하므로, 태그 구조가 디바이스 간에 자동으로 동기화됩니다.

```yaml
# .notology/tag-ontology.yaml 예시
definitions:
  domain/머신러닝:
    label: 머신러닝
    description: "기계학습 관련 주제"
    style:
      color: "#4CAF50"
      borderColor: "#388E3C"
    aliases:
      - ML
      - machine-learning
    children:
      - domain/딥러닝
  domain/딥러닝:
    label: 딥러닝
    description: "심층학습 관련 주제"
    broader: domain/머신러닝
synonyms:
  deep-learning: domain/딥러닝
```

**온톨로지 주요 기능:**
- **자동 동기화** &mdash; 노트에서 사용된 태그가 자동으로 온톨로지에 등록
- **미사용 태그 정리** &mdash; 어떤 노트에서도 사용되지 않는 태그는 세션 시작 시 자동 정리
- **충돌 감지** &mdash; NAS 환경에서 여러 기기가 동시에 태그를 수정해도 병합 전략으로 안전하게 처리
- **커스텀 스타일** &mdash; 태그별 색상, 테두리 색상을 설정 가능 (UI에서 바로 변경)
- **계층 구조** &mdash; `broader`/`children`으로 태그 간 부모-자식 관계 정의 가능

## 폴더 노트 & 컨테이너

### 폴더 노트란?

폴더 노트는 폴더명과 동일한 이름의 마크다운 파일로, 해당 폴더의 **대시보드** 또는 **인덱스** 역할을 합니다. Notology에서 새 폴더를 만들면 폴더 노트가 자동으로 함께 생성됩니다.

### 깊이별 폴더 노트 템플릿

| 깊이 | CSS 클래스 | 설명 | 자동 본문 |
|------|-----------|------|----------|
| **L0** (최상위) | `folder-l0` | 보관함 루트 바로 아래의 대분류 폴더 | 컨테이너 안내 가이드 |
| **L1** (하위) | `folder-l1` | L0 폴더 내부의 소분류 폴더 | 폴더 안내 가이드 |
| **L2+** (심층) | `folder-l2` | 2단계 이상 깊이의 폴더 | 빈 본문 |

### CONTAINER 타입

폴더 노트의 프론트매터 타입은 `CONTAINER`이며, 다음과 같은 기본 상태가 설정됩니다:

```yaml
type: CONTAINER
state:
  workflow: final
  confidence: verified
  maturity: 3
cssclasses:
  - container-type
```

### 활용 전략

```
보관함/
├── 프로젝트/                          (L0 컨테이너)
│   ├── 프로젝트.md                    → 전체 프로젝트 목록, 상태 대시보드
│   ├── 프로젝트 Alpha/               (L1 컨테이너)
│   │   ├── 프로젝트 Alpha.md         → 이 프로젝트의 목표, 타임라인, 링크 모음
│   │   ├── MTG_260208_킥오프.md
│   │   └── NOTE_요구사항 정리.md
│   └── 프로젝트 Beta/
│       ├── 프로젝트 Beta.md
│       └── ...
├── 연구/
│   ├── 연구.md                        → 연구 주제 인덱스, 진행 상황
│   └── ...
└── 아카이브/
    ├── 아카이브.md                     → 완료된 항목 인덱스
    └── ...
```

> **팁:** 폴더 노트에 하위 노트로의 `[[위키링크]]` 목록을 만들면 목차 역할을 합니다. 그래프 뷰에서 폴더 노트는 허브 역할을 하여 구조를 명확하게 보여줍니다.

## 고급 주제

### 보관함 잠금 메커니즘

NAS(Synology Drive 등)나 클라우드 동기화 환경에서 두 기기가 동시에 같은 보관함을 편집하면 데이터가 충돌할 수 있습니다. Notology는 **하트비트 기반 잠금 시스템**으로 이를 방지합니다.

**작동 원리:**

1. 보관함을 열면 `.notology/vault.lock` 파일을 생성하고 잠금을 획득
2. 잠금 파일에는 머신 ID, 호스트명, PID, 앱 버전, 잠금 시각, 하트비트 시각이 기록
3. 15초마다 하트비트를 갱신하여 활성 상태를 알림
4. 다른 기기가 같은 보관함을 열려고 하면:
   - 잠금이 유효하면 (하트비트가 120초 이내) &rarr; **잠금 거부**, 현재 점유자 정보 표시
   - 잠금이 오래되면 (하트비트가 120초 초과) &rarr; **오래된 잠금 감지**, 강제 해제 옵션 제공
5. 강제 해제 시 기존 잠금 파일을 Synology 충돌 스타일로 백업

**잠금 파일 형식:**
```json
{
  "machine_id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "hostname": "DESKTOP-ABC123",
  "pid": 12345,
  "app_version": "1.0.4",
  "locked_at": "2026-02-09T14:30:22Z",
  "heartbeat": "2026-02-09T14:30:37Z"
}
```

### NAS 안전성 (Synology Drive 최적화)

Notology는 Synology Drive 환경을 특별히 고려하여 설계되었습니다.

| 기능 | 설명 | 기술 세부사항 |
|------|------|-------------|
| **Synology 감지** | 보관함이 Synology Drive 동기화 폴더인지 자동 감지 | `.SynologyDrive` 마커 디렉토리, Synology Drive Client 프로세스, 경로 패턴 3가지 전략으로 감지 |
| **원자적 쓰기 (Atomic Write)** | 파일 저장 시 부분 쓰기 방지 | 임시 파일(`*.tmp`)에 먼저 쓰고, 완료 후 원자적 rename. 동기화 중 반쪽 파일이 전파되는 것을 방지 |
| **충돌 감지** | 동기화 충돌 파일 탐지 | Synology의 `(Conflict)` 파일명 패턴 감지 |
| **벌크 동기화 인식** | 대량 동기화 시 UI 안정성 | 다수의 파일이 동시에 변경될 때 UI 업데이트를 일시 중지하여 깜빡임과 성능 저하 방지 |

> **원자적 쓰기가 중요한 이유:** 일반적인 파일 쓰기(`File::create`)는 먼저 파일을 0바이트로 자르고(truncate), 그 다음 내용을 씁니다. Synology Drive가 이 두 단계 사이에 파일을 감지하면 빈 파일(0바이트)이 다른 기기로 동기화될 수 있습니다. 원자적 쓰기는 임시 파일에 완전히 쓴 뒤 이름만 바꾸므로 이 문제가 발생하지 않습니다.

### 백업 시스템

Notology는 파일을 저장할 때마다 이전 버전을 자동으로 백업합니다.

**백업 저장 위치:** `.notology/backups/`

**백업 규칙:**
- 파일을 덮어쓰기 전 자동으로 현재 버전을 백업
- **파일당 최대 5개** 백업 유지 (오래된 것부터 자동 삭제)
- **7일 이상** 된 백업은 자동 정리 (`cleanup_old_backups`)
- 백업 파일명 형식: `원본파일명.YYYY-MM-DDTHH-MM-SS.bak`

**백업 예시:**
```
.notology/backups/
├── NOTE_아이디어.md.2026-02-09T14-30-22.bak
├── NOTE_아이디어.md.2026-02-09T15-10-05.bak
├── MTG_260208_킥오프.md.2026-02-08T16-22-11.bak
└── ...
```

> **팁:** 실수로 노트 내용을 크게 변경했을 때, `.notology/backups/` 폴더에서 이전 버전을 찾아 복원할 수 있습니다.

## 자주 묻는 질문

### 보관함 관련

**Q: 보관함을 여러 개 만들 수 있나요?**
A: 네. Notology를 실행하고 "보관함 열기"로 다른 폴더를 선택하면 새 보관함이 됩니다. 언제든 보관함을 전환할 수 있습니다.

**Q: 기존 마크다운 파일이 있는 폴더를 보관함으로 열 수 있나요?**
A: 네. 기존 `.md` 파일이 있는 폴더를 보관함으로 선택하면, Notology가 자동으로 모든 파일을 인덱싱합니다. 다만, 기존 파일에 Notology 프론트매터가 없으면 일부 기능(태그 필터, 상태 추적 등)이 제한될 수 있습니다.

**Q: 보관함을 다른 컴퓨터로 옮기면 어떻게 되나요?**
A: 보관함 폴더를 통째로 복사하면 됩니다. `.notology/` 디렉토리에 태그 온톨로지, 백업 등 모든 메타데이터가 포함되어 있어 다른 컴퓨터에서 바로 열 수 있습니다.

**Q: "보관함이 잠겨 있습니다" 메시지가 나오면?**
A: 다른 기기에서 같은 보관함을 사용 중입니다. 해당 기기에서 Notology를 종료하세요. 만약 이전 세션이 비정상 종료되어 잠금이 남아 있다면, "강제 잠금 해제" 옵션을 사용하세요.

### 파일 & 동기화

**Q: Notology 파일을 Obsidian이나 다른 앱에서도 열 수 있나요?**
A: 네. 모든 노트는 표준 마크다운 파일이므로 어떤 텍스트 에디터든 열 수 있습니다. 프론트매터는 표준 YAML 형식입니다.

**Q: Dropbox/Google Drive/OneDrive와 호환되나요?**
A: 네. 보관함이 일반 파일 폴더이므로 모든 클라우드 동기화 서비스와 호환됩니다. 다만, 동시 편집 시 보관함 잠금이 클라우드 동기화 지연으로 인해 정확하지 않을 수 있으므로 주의하세요. Synology Drive 환경에서 최적의 안전성이 보장됩니다.

**Q: `.notology/` 폴더를 지워도 되나요?**
A: 지울 수는 있지만 권장하지 않습니다. 백업 히스토리, 태그 온톨로지(색상, 계층 등), 보관함 잠금 정보가 삭제됩니다. 보관함을 다시 열면 기본 `.notology/` 구조가 재생성되지만, 커스텀 설정은 복원되지 않습니다.

### 기능 관련

**Q: 노트에 이미지를 삽입하려면?**
A: `![[이미지파일명.png]]` 위키링크 문법으로 보관함 내 이미지를 인라인 삽입할 수 있습니다.

**Q: 캔버스(SKETCH)와 일반 노트의 차이는?**
A: SKETCH 템플릿으로 만든 노트는 마크다운 에디터 대신 캔버스 에디터가 열립니다. 캔버스 데이터는 프론트매터의 `canvas: true` 플래그로 구분되며, 도형과 연결 데이터가 파일에 저장됩니다.

**Q: 태그를 나중에 변경하거나 삭제할 수 있나요?**
A: 네. 프론트매터에서 직접 태그를 편집하거나, 태그 관리 UI에서 삭제할 수 있습니다. 어떤 노트에서도 사용되지 않는 태그는 다음 세션 시작 시 온톨로지에서 자동 정리됩니다.

**Q: 검색이 느려지면?**
A: Tantivy 인덱스가 손상된 경우일 수 있습니다. 보관함을 닫고 다시 열면 인덱스가 재구성됩니다.
